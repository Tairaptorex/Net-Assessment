Card 1 - Market Research

Summary of research

The range of products researched revealed that while there is significant information available digitally, the information is not presented with group learning in a classroom context as the goal. Based on the strengths and weaknesses identified in the researched products, the following qualities should be considered in the design of our product: Website-based for sharing on a classroom screen (this will also allow for web-based student access on their mobile/class device) Possible interactive elements such as quizzes (Kahoot?) or challenges to solve Focus specifically on a marae visit by school students Clear explanations of tikanga and behaviour expectations for students (for example, unlike an adult, a child would need to be advised not to run noisily around the marae) Simple navigation and function so that the teacher can move through the resource at a pace suited to the students and return to information for easy revision or answering questions Teacher’s checklist for preparedness

Card 3 - Prototype Evaluation

Evaluation of Initial Designs

Layout Version

Pros: Familiar for end users, simple to set up, definitions are easy to access, image embedding is easy, menu slides away for minimalism, responsive layout is great for phones. I’ve built many of these.

Cons: Not doable in typical website builders.

Running Blog Version

Pros: Can be set up in sites. Easy to use. Minimalist.

Cons: Not responsive unless programmed from scratch. Lots of context risks muddling navigation.

Dropdown Version

Pros: Very Minimalist. Easy to navigate. Load time can be sped up with force loading.

Cons: Website can seem unfinished. Very hard to make aesthetic. Lots of user input required. Information can be concealed. Not doable in typical website builders.

Persistent Navigation Version

Pros: Very Professional standard. Ease of navigation. Responsive for laptops. Information is easy to access. High aesthetic potential. Very responsive.

Cons: Not doable in typical website builders. Very Technical setup. Navigation benefits must be sacrificed for phone responsiveness.
